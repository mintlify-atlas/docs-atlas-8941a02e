Access comprehensive audit logs for compliance, security, and traceability purposes.

## Overview

The SGRH audit system automatically records all significant actions performed in the system, providing:

- **Complete traceability**: Who did what, when, and from where
- **Change tracking**: Before/after values for all modifications
- **Compliance**: Meet regulatory and audit requirements
- **Security**: Detect unauthorized access and suspicious activities

## Audit Event Object

An audit event represents a single recorded action in the system.

### Attributes

<ResponseField name="auditoriaEventoId" type="number">
  Unique identifier for the audit event
</ResponseField>

<ResponseField name="fechaUtc" type="string">
  Timestamp when the event occurred (UTC)
</ResponseField>

<ResponseField name="usuarioId" type="number">
  ID of the user who performed the action
</ResponseField>

<ResponseField name="rol" type="string">
  User's role at the time of the action (snapshot)
</ResponseField>

<ResponseField name="usernameSnapshot" type="string">
  Username at the time of the action (snapshot)
</ResponseField>

<ResponseField name="accion" type="string">
  Action performed: `CREATE`, `UPDATE`, `DELETE`, `CONFIRM`, `CANCEL`, etc.
</ResponseField>

<ResponseField name="modulo" type="string">
  System module: `Reservas`, `Clientes`, `Habitaciones`, `Tarifas`, etc.
</ResponseField>

<ResponseField name="entidad" type="string">
  Entity type affected: `Reserva`, `Cliente`, `Habitacion`, etc.
</ResponseField>

<ResponseField name="entidadId" type="string">
  ID of the affected entity
</ResponseField>

<ResponseField name="requestId" type="string">
  Unique request ID for correlation (GUID)
</ResponseField>

<ResponseField name="ipOrigen" type="string">
  Source IP address
</ResponseField>

<ResponseField name="userAgent" type="string">
  User agent string (browser/client information)
</ResponseField>

<ResponseField name="descripcion" type="string">
  Human-readable description of the action
</ResponseField>

<ResponseField name="detalles" type="array">
  Field-level changes (for UPDATE actions)
  <ResponseField name="campo" type="string">
    Field name that changed
  </ResponseField>
  <ResponseField name="valorAnterior" type="string">
    Previous value
  </ResponseField>
  <ResponseField name="valorNuevo" type="string">
    New value
  </ResponseField>
</ResponseField>

## List Audit Events

Retrieve a paginated list of audit events with filtering.

### Endpoint

```http
GET /api/auditoria
```

### Query Parameters

<ParamField query="pageNumber" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="pageSize" type="number" default="50">
  Number of items per page (max 500)
</ParamField>

<ParamField query="fechaDesde" type="string">
  Filter events from this date (ISO 8601)
</ParamField>

<ParamField query="fechaHasta" type="string">
  Filter events until this date (ISO 8601)
</ParamField>

<ParamField query="usuarioId" type="number">
  Filter by user ID
</ParamField>

<ParamField query="accion" type="string">
  Filter by action type
</ParamField>

<ParamField query="modulo" type="string">
  Filter by module name
</ParamField>

<ParamField query="entidad" type="string">
  Filter by entity type
</ParamField>

<ParamField query="entidadId" type="string">
  Filter by specific entity ID
</ParamField>

<ParamField query="requestId" type="string">
  Filter by request ID (for correlation)
</ParamField>

<ParamField query="ipOrigen" type="string">
  Filter by source IP address
</ParamField>

### Request Example

<CodeGroup>
```bash cURL
curl -X GET "https://api.sgrh.com/api/auditoria?modulo=Reservas&accion=UPDATE&fechaDesde=2024-03-01" \
  -H "Authorization: Bearer {token}"
```

```csharp C#
var query = "?modulo=Reservas&accion=UPDATE&fechaDesde=2024-03-01";
var response = await client.GetAsync(
    $"https://api.sgrh.com/api/auditoria{query}"
);
```
</CodeGroup>

## Common Action Types

Standard action types recorded in the audit system:

| Action | Description |
|--------|-------------|
| `CREATE` | New entity created |
| `UPDATE` | Entity updated |
| `DELETE` | Entity deleted |
| `CONFIRM` | Reservation confirmed |
| `CANCEL` | Reservation cancelled |
| `CHECKIN` | Check-in performed |
| `CHECKOUT` | Check-out performed |
| `LOGIN` | User logged in |
| `LOGOUT` | User logged out |
| `EXPORT` | Report exported |

## Authorization

<Warning>
**Role Requirements**

Audit endpoints are only accessible to users with the `Administrador` role. Recepcionistas cannot access audit logs.
</Warning>

## Data Retention

Audit logs are retained according to the following policy:

- **Active logs**: Last 90 days (queryable via API)
- **Archived logs**: 91 days to 7 years (available on request)
- **Permanent retention**: Security incidents and compliance events
